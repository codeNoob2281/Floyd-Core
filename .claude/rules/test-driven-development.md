# Test-Driven Development(TDD)

All feature development and bug fixes must follow the **TDD** workflow:

1. **Red** — Write a failing test that defines the expected behavior
2. **Green** — Write the minimum code to make the test pass
3. **Refactor** — Clean up the code while keeping tests green

## Rules

- Write tests **before** writing implementation code
- Each test should cover one specific behavior or scenario
- Run the test suite after every change to ensure no regressions
- Aim for meaningful coverage of success paths, edge cases, and error handling
