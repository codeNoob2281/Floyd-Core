# Floyd-Core 框架概述

## 简介

Floyd-Core 是一个轻量级的 Minecraft 插件开发框架，基于 Spring Framework 和 PaperMC API 构建。它旨在通过现代 Java 实践简化插件开发流程，提供丰富的功能模块和优雅的开发体验。

## 核心特性

### 🍃 轻量级设计
- 简洁直观的 API 设计
- 低学习曲线，易于上手
- 最小化依赖，减少冲突

### 🌱 Spring 集成
- 完整的 Spring Framework 支持
- 依赖注入和 Bean 管理
- AOP 面向切面编程支持
- 配置类管理

### 🔒 权限系统
- 基于注解的权限检查
- AOP 实现的无侵入式设计
- 自动玩家权限验证
- 可自定义错误消息

### 📦 物品序列化
- ItemStack 序列化/反序列化工具
- Base64 编码安全存储
- 支持数据库和配置文件存储

### 📝 日志系统
- 多级别日志支持（DEBUG/INFO/WARN/ERROR）
- 控制台和文件双重输出
- 可配置的日志行为

### 🗄️ 数据库支持
- 内置数据库管理模块
- 支持多种数据库类型
- SQL 语法构建器
- 数据备份功能

### 🌍 国际化支持
- 多语言消息管理
- YAML 格式语言文件
- 动态语言切换
- 消息占位符支持

### ⚙️ 配置管理
- 插件设置管理器
- 热重载支持
- 类型安全的配置访问
- 默认值管理

## 技术栈

- **Java**: 21+
- **Spring Framework**: 6.2.7
- **PaperMC API**: 1.21.11
- **Lombok**: 1.18.30
- **AspectJ**: 1.9.7

## 架构设计

Floyd-Core 采用模块化设计，各功能模块相互独立又协同工作：

```
FloydPlugin (核心基类)
├── Spring 容器管理
├── 生命周期管理
├── 配置管理
└── 模块初始化
    ├── Command Module (命令系统)
    ├── Database Module (数据库模块)
    ├── I18n Module (国际化模块)
    ├── Permission Module (权限模块)
    ├── Logging Module (日志模块)
    ├── Settings Module (配置模块)
    └── Inventory IO Module (物品序列化)
```

## 适用场景

- 中大型 Minecraft 插件开发
- 需要复杂业务逻辑的插件
- 需要良好架构设计的团队项目
- 追求代码质量和可维护性的开发者
